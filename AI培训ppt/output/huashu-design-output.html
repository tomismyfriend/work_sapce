LLM 核心原理

🎯

下一个词预测器

LLM 的本质是一个超级概率模型：
给定前文，预测下一个最可能的 Token

"今天天气真" → P("好") = 0.72
                    P("热") = 0.15
                    P("冷") = 0.08

三阶段训练

1

预训练 (Pre-training)

海量文本数据，学习语言规律与世界知识

2

监督微调 (SFT)

高质量问答对，学会遵循指令回答问题

3

强化学习 (RLHF)

人类偏好反馈，对齐价值观与输出质量

07

Token — AI 的计量单位

什么是 Token？

Token 是 LLM 处理文本的最小单元。
英文中 1 Token ≈ 4 字符 / 0.75 个单词
中文中 1 个汉字 ≈ 1-2 个 Token

// Tokenization 示例
"Hello World" → ["Hello", " World"] = 2 tokens
"人工智能"   → ["人工", "智能"] = 2 tokens

成本对比 (每百万 Token)

模型 输入 输出
GPT-4o $2.50 $10.00
Claude 3.5 $3.00 $15.00
Qwen-Max ¥2.00 ¥6.00
DeepSeek-V3 ¥1.00 ¥2.00

记忆 — Agent 的存储系统

⚡

短期记忆

Working Memory

  • • 当前对话上下文
  • • 中间推理步骤
  • • 工具调用结果
  • • 受 Token 窗口限制
容量：~128K tokens
💾

长期记忆

Long-term Memory

  • • 向量数据库存储
  • • 历史对话摘要
  • • 用户偏好画像
  • • 语义相似度检索
容量：理论上无限
📐

程序记忆

Procedural Memory

  • • 系统 Prompt 模板
  • • 工具使用规范
  • • 工作流定义
  • • 代码中的硬编码逻辑
容量：由代码定义

Agent Loop — 核心执行循环

async function agentLoop(task, tools) { let messages = [ { role: "system", content: SYSTEM_PROMPT }, { role: "user", content: task } ]; while (true) { // 1. LLM 思考与决策 const response = await llm.chat(messages); // 2. 检查是否有工具调用 if (response.tool_calls?.length) { for (const call of response.tool_calls) { // 3. 执行工具 const result = await executeTool(call); messages.push({ role: "tool", content: result }); } continue; // 回到循环开头 } // 4. 没有工具调用 → 最终回答 return response.content; } }

执行流程

1

接收任务

构建初始消息列表

2

LLM 决策

推理下一步行动或直接回答

3

执行工具

调用函数并将结果喂回 LLM

4

循环或终止

继续迭代直到任务完成
