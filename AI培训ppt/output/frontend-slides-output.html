04 — LLM

什么是 LLM？

大语言模型 — 本质是一个下一个词预测器

三阶段训练

1. 预训练
万亿级文本，预测下一个词
2. 监督微调 (SFT)
高质量问答对，学会"回答问题"
3. 偏好对齐 (RLHF)
人类偏好排序，遵循复杂指令

三个关键能力

能力对 Agent 的意义
通用推理充当决策大脑
上下文学习动态定义行为模式
指令遵循可靠执行工具调用
LLM 本身只会"生成文本" — 没有手、没有脚、没有记忆

05 — TOKEN

什么是 Token？

Token = 词表中一个条目的 ID = 大模型处理文本的最小单位

Token 计数示例

文本类型示例Token 数
常见汉字"你好"2
英文单词"hello"1
长英文词"unbelievable"3
代码"def fact(n):"6

控制成本

策略效果
简单任务用小模型-90%
精简 Prompt-75%
缓存重复请求零费用
Batch API-50%
Token 是 AI 时代的"计量单位"。选对模型 + 精简输入 + 缓存复用 = 成本降低 50-80%

06 — LIMITATION

为什么 LLM 不能直接"操作"？

LLM = f(text_in) → text_out    // 就这样。没有 IO，没有网络，没有数据库
限制原因影响
无 IO 能力只有矩阵运算不能读写文件
无状态每次调用独立不记住上次对话
无执行环境运行在 GPU 上不能运行代码
单次输出一次只生成一段文本不能分步执行
知识截止训练数据有截止日不知道最新信息
LLM 提供智能，Agent 提供执行力。没有 Agent 框架，LLM 只是"纸上谈兵"的天才

11 — AGENT LOOP

循环执行 The Core

def run_agent(user_input):
  messages = [system_prompt, user_input]

  while True:
    response = llm.chat(messages, tools)

    if response.tool_calls:
      for tc in response.tool_calls:
        result = execute_tool(tc)
        messages.append(result)
    else:
      return response.content

13 — COMPLETION

任务如何完成？

核心答案：不是代码判断的，是 LLM 自己决定的

输出 A: 调用工具

tool_calls 不为空

→ 继续循环

输出 B: 直接回复

tool_calls 为空

→ 任务完成

安全防护

最大步数
max_steps 防止无限循环
结果验证
validate 检查完整性
强制继续
不完整则追加提示
模型越强，判断越准确；模型越弱，越容易过早停止或无限循环

14 — VALIDATION

结果验证 三种方式

01

程序化验证

最可靠 • 最低成本
  • 数学计算 → 代码验证
  • 代码生成 → 运行测试
  • API 调用 → HTTP 状态码
02

LLM-as-Judge

中等可靠 • 中等成本
  • 适用于开放式任务
  • 文案、翻译、总结
  • LLM 自我评估质量
03

交叉验证

最严谨 • 最高成本
  • 医疗、法律、金融
  • 模型 A 执行
  • 模型 B 验证 + C 比较
越重要的任务越需要多重验证

15 — IMPLEMENTATION

50 行代码实现 Agent

import openai, json
client = openai.OpenAI()

tools = [{"type": "function",
  "function": {"name": "get_weather",
    "description": "获取天气",
    "parameters": {...}}}]]

def execute_tool(name, args):
  if name == "get_weather":
    return json.dumps({"weather": "小雨"})

def run_agent(user_input, max_steps=10):
  messages = [{"role": "system", ...},
            {"role": "user", "content": user_input}]
  for step in range(max_steps):
    resp = client.chat.completions.create(
      model="gpt-4o", messages=messages, tools=tools)
    msg = resp.choices[0].message
    if msg.tool_calls:
      messages.append(msg)
      for tc in msg.tool_calls:
        result = execute_tool(...)
        messages.append({"role": "tool", ...})
    else:
      return msg.content

关键洞察

LLM 不执行操作
只输出"我想调用 get_weather"
你的代码执行
execute_tool 是你写的函数
Agent = 胶水代码
连接 LLM 决策和真实工具

16 — SELF-CORRECTION

自我纠错能力

实际案例

R1 run_tests() → 3 tests failed
R2 分析原因 → 缺少环境变量
R3 set_env("AUTH_KEY") → 成功
R4 run_tests() → All passed ✓

三种反思模式

① 结果验证
执行 → 检查 → 不符合则修正
② 自我评估
生成 → 评估质量 → 低分重新生成
③ 多路径探索
多种解法 → 比较 → 选择最优
