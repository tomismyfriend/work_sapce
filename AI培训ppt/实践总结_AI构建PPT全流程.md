# AI 构建 PPT 实践总结

> 实践时间：2026年6月23日
> 实践目标：从零开始，用 AI 完成"内容创作 → PPT 生成"全流程
> 最终主题：AI Agent 开发与智能编程工具实战

---

## 一、实践背景

### 目标

为技术开发者准备一场约 30 分钟的 AI 培训 PPT，覆盖 AI Agent 的核心原理和 AI 编程工具的实战技巧。

### 工具链

```
内容创作阶段：AI 对话（OpenCode + 大模型）
  ↓ 输出 Markdown 文稿
PPT 生成阶段：4 个 PPT Skills 并行生成
  ├── frontend-slides    → 单个 HTML 演示文稿
  ├── ppt-svg-generator  → 15 个 SVG（可导入 PPT）
  ├── open-slide         → React 幻灯片项目
  └── huashu-design      → 单个 HTML 演示文稿
```

---

## 二、内容创作阶段

### 2.1 主题确认（约 5 分钟）

通过 AI 提问确认三个关键参数：

| 维度 | 选择 | 理由 |
|------|------|------|
| 受众 | 技术开发者 | 有编程基础的工程师 |
| 主题 | AI Agent + AI 编程工具 | 两个方向结合，逻辑连贯 |
| 语言 | 中文 | 技术术语保留英文 |

AI 推荐主题：**「AI Agent 开发与智能编程工具实战」**——先理解 Agent 原理，再用 AI 工具高效开发。

### 2.2 初版内容生成（约 10 分钟）

AI 一次性生成 18 页 Slides 的完整内容，包括：
- 每页的标题、要点、代码示例、对比表格、架构图
- 时间分配建议（2+8+8+8+4 分钟）

### 2.3 迭代深化（约 40 分钟，6 轮对话）

| 轮次 | 用户要求 | AI 执行 | 页数变化 |
|------|---------|---------|---------|
| 第 1 轮 | 深入解析 Agent 与大模型的关系、为什么能独立完成任务 | 新增 4 页（LLM 本质、Agent 关系、Agent Loop、Function Calling） | 18 → 22 |
| 第 2 轮 | 进一步深入，从最基础原理讲起 | 新增 4 页（Transformer、训练流水线、涌现能力、幻觉问题） | 22 → 28 |
| 第 3 轮 | 代码上怎么实现 Agent，怎么判断答复正确 | 新增 3 页（50 行代码实现、任务完成判断、结果验证） | 28 → 31 |
| 第 4 轮 | 大模型训练的回归过程 | 新增 3 页（前向传播、反向传播、分布式训练） | 31 → 34 |
| 第 5 轮 | 补充自监督学习、完整训练流程 | 新增 3 页（自监督学习、SFT/DPO 对齐、完整流水线） | 34 → 37 |
| 第 6 轮 | 补充 Token 概念和成本控制 | 新增 2 页（Token 原理、计费与省钱策略） | 37 → 39 |

### 2.4 结构优化（约 15 分钟）

用户提出"精简内容，改为总-分结构"：

```
优化前（39 页，线性展开）:
  概念 → 原理 → 原理 → 原理 → ... → 实战

优化后（24 页，总-分结构）:
  总：整体认知 Agent（4 页）
  分：逐个展开核心原理（14 页）
  实战：AI 编程工具（6 页）
```

**删除的过于专业内容：**
- Transformer 注意力机制的数学细节
- 前向传播/反向传播公式推导
- 分布式训练（数据并行/张量并行）
- 完整训练流水线（数据工程/评估迭代）

### 2.5 补充 MCP 协议和感知能力（约 10 分钟）

| 新增内容 | 页数 | 核心要点 |
|---------|------|---------|
| MCP 协议 | +1 页 | USB-C 类比，N×M → N+M 问题 |
| 感知能力扩展 | 扩展 1 页 | 5 种感知方式、固定/动态/混合模式 |
| 记忆系统扩展 | 扩展 1 页 | 三种记忆的实现原理、向量检索、生命周期 |
| 系统架构总览 | +1 页 | 五大特征对应五大模块 |

### 2.6 最终文稿

```
文件：PPT内容_AI_Agent开发与智能编程工具实战.md
页数：27 页 Slides
行数：约 1,660 行
大小：约 65KB

结构：
  第一部分（总）：Slide 1-5  — 整体认知 AI Agent
  第二部分（分）：Slide 6-19 — 逐个展开核心原理
  第三部分（实战）：Slide 20-27 — AI 编程工具与总结
```

## 三、PPT 生成阶段

### 3.1 Skill 探索（约 10 分钟）

AI 自动探索 4 个 Skill 目录，读取 README、SKILL.md、命令定义、模板文件、样式配置，输出详细对比表：

| Skill | 类型 | 输入 | 输出 | 特点 |
|-------|------|------|------|------|
| frontend-slides | Claude Code Skill | Markdown/自然语言 | 单个 HTML | 零依赖，动画丰富，12 种风格预设 |
| ppt-svg-generator | OpenCode Skill | Markdown 文件 | SVG 文件 | PPT 兼容，5 种中文风格，可导入 PPT 编辑 |
| open-slide | React 框架 | React TSX 组件 | React 项目 | 1920×1080 画布，Vite 热更新 |
| huashu-design | 通用设计 Skill | 自然语言 | HTML/MP4/PDF | 40 种风格，支持动画导出 |

### 3.2 中间内容准备（约 3 分钟）

创建精简版中间文件 `output/slides-condensed.md`，将 27 页详细内容压缩为适合视觉演示的信息密度（每页只保留核心要点）。

### 3.3 四路并行生成（约 15 分钟）

4 个 Task Agent 并行执行，各自独立生成：

#### ① frontend-slides → HTML 演示文稿

```
输出：output/frontend-slides-output.html（80KB）
风格：Neon Cyber（深蓝 #0a0f1c + 霓虹青 #00ffcc + 品红 #ff00aa）
字体：Orbitron（标题）+ Rajdhani（正文）
页数：20 页
特性：
  - 粒子动画背景（Canvas 绘制）
  - 霓虹发光效果（box-shadow）
  - 网格图案背景
  - 键盘/触摸/滚轮导航
  - Intersection Observer 入场动画
  - prefers-reduced-motion 支持
查看方式：浏览器直接打开 HTML 文件
```

#### ② ppt-svg-generator → SVG 幻灯片

```
输出：output/ppt-svg-output/（15 个 SVG 文件）
风格：科技暗黑（暗色 #0D0D0D + 荧光青 #00F5D4 + 科技紫 #7B61FF）
字体：Microsoft YaHei, SimHei, PingFang SC
页数：15 页
特性：
  - 遵循 PPT 兼容性 5 规则
  - <path> 贝塞尔曲线圆角（不用 rx）
  - #RRGGBB 颜色 + fill-opacity（不用 rgba）
  - 1920×1080 标准比例
  - 中文字体兼容 Windows/Mac
查看方式：
  - 浏览器直接打开 SVG 预览
  - 导入 PowerPoint → 右键"转换为形状" → 可二次编辑
```

#### ③ open-slide → React 项目

```
输出：output/ai-agent-slides/（完整 React 项目）
风格：自定义暗色主题（#0a0f1c + #00ffcc）
字体：Orbitron + Rajdhani + JetBrains Mono
页数：15 页
特性：
  - 1920×1080 固定画布
  - DesignSystem 导出（palette/fonts/typeScale）
  - Vite 热更新开发服务器
  - 已构建 dist/ 目录（可直接预览）
查看方式：
  - npm run dev → 开发模式（热更新）
  - npm run preview → 预览构建版本（http://localhost:4173）
  - 直接打开 dist/index.html
```

#### ④ huashu-design → HTML 演示文稿

```
输出：output/huashu-design-output.html（49KB）
风格：Build Studio（白底 #FAFAFA + 橙色 #FF4500 + 蓝色 #2563EB）
字体：DM Sans + IBM Plex Mono
页数：15 页
特性：
  - 1920×1080 画布 + CSS transform 自适应缩放
  - 橙色强调线 + 大号章节编号
  - 卡片式布局，编辑风格
  - 键盘/触摸导航 + 进度条 + 幻灯片计数器
查看方式：浏览器直接打开 HTML 文件
```

## 六、经验与建议

### 内容创作阶段

**✅ 有效做法：**
1. **先确认受众和场景** — 决定了内容深度和语言风格
2. **先生成再迭代** — 让 AI 先出一版完整内容，再逐步深化，比从零开始高效得多
3. **用"我不太理解"驱动深化** — 当你觉得某个概念不清楚时，说明观众也会不清楚，这正是需要深化的地方
4. **适时精简** — 内容过多时果断删减，保持"总-分"结构清晰
5. **用类比解释概念** — "USB-C 接口"解释 MCP、"被关在房间里的天才"解释 LLM 限制

**❌ 应避免：**
1. 一开始就追求完美 — 先有骨架再长肉
2. 过度深入底层原理 — 培训 PPT 不是论文，概念通顺比数学严谨更重要
3. 忽略信息密度 — 每页 Slide 内容不宜过多，宁可多分几页

### PPT 生成阶段

**✅ 有效做法：**
1. **创建中间文件** — 将详细文稿压缩为适合视觉演示的密度，再生成 PPT
2. **并行生成** — 4 个 Skill 同时执行，15 分钟得到 4 套不同风格的 PPT
3. **根据场景选择 Skill：**
   - 在线分享/快速演示 → frontend-slides 或 huashu-design（双击打开）
   - 正式汇报/需要编辑 → ppt-svg-generator（导入 PPT）
   - 技术团队/需要定制 → open-slide（React 项目）

### 选择合适的 Skill

| 你的需求 | 推荐 Skill | 理由 |
|---------|-----------|------|
| 快速出一版好看的 PPT | frontend-slides | 零依赖，动画丰富，双击打开 |
| 需要在 PPT 中二次编辑 | ppt-svg-generator | SVG 导入 PPT 后可转换为形状编辑 |
| 团队都是开发者，想定制 | open-slide | React 组件，版本控制友好 |
| 需要多种输出格式 | huashu-design | 支持 HTML/MP4/GIF/PDF/PPTX |
| 中文内容为主 | ppt-svg-generator | 中文字体优化最好 |
